# Plan

Create a detailed implementation plan for a goal or feature.

## Arguments
$ARGUMENTS - Description of what to plan

## Instructions

1. Invoke the **Planner Agent** (see `agents/planner.md`) mindset
2. Understand the goal described in: $ARGUMENTS
3. If the goal is vague, ask clarifying questions before planning
4. Research the current codebase/project state relevant to the goal
5. Decompose the goal using the Task Decomposition Framework (see `.docs/ways-of-working.md`)
6. Produce a plan with:
   - Overview of the approach
   - Phased task breakdown with dependencies
   - Agent assignments for each task
   - Risk identification and mitigations
   - Estimated complexity per task
   - Parallelization opportunities
7. Present the plan to the user for approval
8. On approval, create the task list and begin execution
